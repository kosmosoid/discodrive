# DiscoDrive

[English](../README.md) · [Deutsch](README.de.md) · [Українська](README.uk.md) · [Français](README.fr.md) · [Español](README.es.md) · [**Русский**](README.ru.md) · [Српски](README.sr.md)

**Ваше личное облако. Один сервер, один логин — файлы, календари, контакты, задачи, музыка и книги.**

DiscoDrive — это self-hosted-замена набору подписок и разрозненных утилит, которые обычно приходится склеивать, чтобы уйти с iCloud, Google или Dropbox. Вместо того чтобы держать File Browser, Radicale, Subsonic-сервер *и* Calibre-web — каждый со своим трудностями и особенностями — Вы запускаете одно приложение. Всё, что описано ниже, живёт в одном аккаунте, на Вашем оборудовании, под Вашим контролем.

Файлы хранятся как обычные файлы в обычных папках на диске. Если вдруг понадобится переехать с DiscoDrive на другое хранилище, Вы просто откроете папку и найдёте свои данные ровно там, где их оставили. Никакого проприетарного хранилища фрагментами, никакой базы, которую невозможно прочитать, никакой привязки к одному поставщику (vendor lock-in).

Кроссплатформенность заложена в основу: Apple, Windows, Linux и Android.

## Что внутри

### 📁 Файлы и синхронизация

- Быстрый современный **файловый менеджер в браузере** — загрузка, скачивание, создание папок, переименование, перемещение, с перетаскиванием (drag-and-drop) целых файлов *и* папок.
- **Версии** — у каждого файла хранится недавняя история, можно вернуться к прежней версии после неудачной правки.
- **Корзина** — удалённые файлы попадают в корзину, откуда их можно восстановить.
- **Общий доступ** — публичные ссылки: с истечением срока, с паролем, только на чтение.
- **Подключается как диск** — открывайте хранилище прямо из Finder, Проводника Windows, файлового менеджера Linux или файлового приложения Android по WebDAV. Ничего устанавливать не нужно.
- **Синхронизация на компьютере** — лёгкое приложение держит реальную папку на компьютере синхронизированной с сервером, как Dropbox.
- **Данные не теряются** — если один и тот же файл изменился в двух местах сразу, сервер сохранит обе версии как конфликтные копии.

### 🔒 Зашифрованный сейф

- **Приватная папка со сквозным (E2E) шифрованием** для самых чувствительных файлов.
- Файлы шифруются **на Вашем устройстве** ещё до отправки — сервер хранит их только в зашифрованном виде.
- Открывается прямо в браузере или в клиентском приложении (расшифровка происходит локально), а если Вы забыли пароль — сейф разблокируется кодом восстановления.
- Построен на **открытом формате, совместимом с [Cryptomator](https://cryptomator.org)** — если понадобится открыть сейф отдельно или перейти на другой сервис, его можно расшифровать в Cryptomator или другими сторонними приложениями.

### 📅 Календарь, контакты и задачи

- Полноценные **календари, адресные книги и напоминания** — события, контакты и дела в одном месте.
- **Нативно на Apple** — Вы добавляете аккаунт в системных настройках iOS/macOS, и штатные Календарь, Контакты и Напоминания просто работают, без отдельного приложения.
- **На всём остальном** — работает с DAVx⁵ на Android и с Thunderbird, Evolution или eM Client на компьютере.
- Удобный **веб-интерфейс** для создания и редактирования событий, контактов и задач.
- **Делитесь календарём или адресной книгой** с близкими.

### 🎵 Музыка — Ваш личный стриминг

- Превращает Вашу коллекцию музыки в личный стриминговый сервис, доступный откуда угодно.
- **Работает с привычными приложениями** — любой OpenSubsonic-совместимый плеер (Amperfy, Feishin и многие другие) подключается без настройки, с текстами песен, очередью и поиском.
- **Интернет-радио** — добавляйте любимые станции и слушайте их через те же приложения.
- **Подкасты** — подписывайтесь на ленты, скачивайте и слушайте эпизоды, продолжайте с того места, где остановились, по закладкам.
- **Встроенный редактор тегов** — редактируйте названия, исполнителей, альбомы, жанры и обложки прямо в браузере: по одному треку или целой папкой сразу.

### 📚 Книги — Ваша личная библиотека

- Личная библиотека для электронных книг и комиксов, готовая к чтению на любом устройстве.
- **Читается на любом устройстве** — каталог по стандарту OPDS, к которому напрямую подключаются KOReader, PocketBook, Marvin и другие приложения для чтения (а через браузер — вообще любое устройство).
- **Все нужные форматы** — EPUB, FB2, PDF, MOBI, CBZ и CBR, с миниатюрами обложек и поиском по всему каталогу.
- **Синхронизация прогресса чтения** — начните книгу на одном устройстве и продолжите ровно с того же места на другом.
- **Редактор метаданных** — редактируйте названия, авторов, серии, теги и описания в браузере: по одной книге или сразу по всей папке.

### 🛡️ Аккаунты и безопасность

- **Двухфакторная аутентификация** TOTP через любое приложение-аутентификатор плюс одноразовые резервные коды.
- **Passkeys** — вход по Face ID, Touch ID или аппаратному ключу, вообще без пароля.
- **Журнал безопасности**.
- **Уведомления по электронной почте** о важных событиях безопасности и аккаунта.
- **Встроенная защита от подбора паролей**.
- **Несколько пользователей** с персональными квотами.
- **Пароли для приложений** — чтобы подключать файловые, музыкальные и книжные приложения, не раскрывая основной пароль.
- **TLS везде** по умолчанию.

### 🌍 Ваше, везде и на Вашем языке

- Совместимо с операционными системами Apple, Windows, Linux и Android.
- В данный момент доступны **7 языков** интерфейса: английский, немецкий, украинский, французский, испанский, русский и сербский.

## Установка

DiscoDrive — это один сервер плюс PostgreSQL. Перед ним можно поставить nginx — для TLS и быстрой отдачи файлов через X-Accel. Ниже два способа развернуть всё это. Большинству подойдёт **Вариант 1**.

> **Перед стартом необходимо подготовить два секретных ключа.** Сервер не запустится с временными значениями из `.env.example`. Сгенерируйте их один раз и держите неизменными:
>
> - `JWT_SECRET` — `openssl rand -base64 48`;
> - `SETTINGS_ENCRYPTION_KEY` — `openssl rand -hex 16`.
>
> И помните: приложениям Apple (CalDAV/CardDAV) нужен **HTTPS** — по незашифрованному HTTP они не работают.

### Вариант 1. Docker, с клонированием репозитория (рекомендуется)

Самый простой путь: одна команда `docker compose up` поднимает сервер, базу данных и nginx с TLS.

**Нужно:** git, Docker и Docker Compose.

```sh
git clone https://github.com/kosmosoid/discodrive.git
cd discodrive
cp .env.example .env
# отредактируйте .env: задайте JWT_SECRET, SETTINGS_ENCRYPTION_KEY, BASE_DOMAIN,
# POSTGRES_PASSWORD и, при желании, остальное
docker compose up -d
```

Откройте **https://server_address:8443** (или http://server_address:8080) — при первом запуске откроется создание администратора. Укажите почту и пароль, и можно работать. Учтите, что если вы планируете настроить отправку почтовых сообщений от сервиса, то необходимо использовать действующие почтовые адреса для имён пользователей (в том числе администратора), в противном случае сообщения не смогут быть доставлены.

Как это устроено:

- Файлы хранятся в `/data`.
- TLS-сертификаты в `deploy/nginx/certs/` - Необходимо создать и положить **настоящий или самоподписанный сертификат** (как сгенерировать — в Варианте 2) и переведите в конфигурации nginx 80-й порт на перенаправление в HTTPS.

Чтобы остановить: `docker compose down` (данные сохранятся). Чтобы обновить после `git pull`: `docker compose up -d --build`.

### Вариант 2. Отдельными компонентами (сборка из исходного кода)

Если Вы не хотите использовать Docker — соберите исполняемый файл сами и запустите его рядом с обычным PostgreSQL. Исполняемый файл самодостаточен - в него включены все компоненты и составляющие.

**Для сборки необходимы:** Go 1.25+, Node.js 22+, PostgreSQL 16+.

**1. Соберите веб-интерфейс** (Nuxt → статические файлы в `web/dist`):

```sh
cd web
npm install
npm run generate
cd ..
```

**2. Соберите сервер** (исполняемый файл вберёт в себя `web/dist` и миграции):

```sh
CGO_ENABLED=0 go build -trimpath -o discodrive ./cmd/server
```

**3. Поднимите PostgreSQL** и создайте базу данных с пользователем:

```sh
createuser disco --pwprompt
createdb discodrive --owner disco
```

**4. Задайте переменные окружения** (полный список — в `.env.example`):

```sh
export DATABASE_URL="postgres://disco:ПАРОЛЬ@localhost:5432/discodrive?sslmode=disable"
export JWT_SECRET="$(openssl rand -base64 48)"
export SETTINGS_ENCRYPTION_KEY="$(openssl rand -hex 16)"
export BASE_DOMAIN="example.com"
export STORAGE_ROOT="/var/lib/discodrive/data"   # каталог должен существовать и быть доступен на запись
export APP_PORT="8080"
export XACCEL_ENABLED="false"                     # без nginx сервер раздаёт файлы сам
```

**5. Запустите**:

```sh
./discodrive
```

Откройте `http://server_address:8080` и создайте администратора.

**nginx (необязательно).** Нужен для HTTPS и для быстрой отдачи файлов через X-Accel. Возьмите за основу `deploy/nginx/default.conf`, пропишите свои TLS-сертификаты и установите `XACCEL_ENABLED=true` — тогда сервер отдаёт файлы заголовком `X-Accel-Redirect`, а само тело файлов раздаёт nginx (расположение `/data/`, значение параметра `STORAGE_ROOT`).

TLS-сертификаты возьмите у Let's Encrypt (`certbot`) или сгенерируйте самоподписанные:

```sh
openssl req -x509 -newkey rsa:2048 -nodes -days 825 \
  -keyout deploy/nginx/certs/dev-key.pem \
  -out deploy/nginx/certs/dev.pem -subj "/CN=localhost"
```

**Автозапуск через systemd (необязательно).** Пример модуля `/etc/systemd/system/discodrive.service`:

```ini
[Unit]
Description=DiscoDrive
After=network.target postgresql.service

[Service]
ExecStart=/usr/local/bin/discodrive
EnvironmentFile=/etc/discodrive.env
Restart=on-failure
User=discodrive

[Install]
WantedBy=multi-user.target
```

Переменные из шага 4 сложите в `/etc/discodrive.env` (формат `KEY=value`, по одной на строку), затем выполните:

```sh
sudo systemctl daemon-reload
sudo systemctl enable --now discodrive
```
