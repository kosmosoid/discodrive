# DiscoDrive

[English](../README.md) · [Deutsch](README.de.md) · [**Українська**](README.uk.md) · [Français](README.fr.md) · [Español](README.es.md) · [Русский](README.ru.md) · [Српски](README.sr.md)

**Твоя власна хмара. Один сервер, один логін — файли, календарі, контакти, задачі, музика та книжки.**

DiscoDrive — це self-hosted заміна набору підписок і зоопарку утиліт, які зазвичай доводиться склеювати, щоб перейти з iCloud, Google чи Dropbox. Замість того щоб тримати File Browser, Radicale, Subsonic-сервер *та* Calibre-web — кожен зі своїм логіном і своїми граблями — ти запускаєш одну річ. Усе, що описано нижче, живе в одному акаунті, на твоєму залізі, під твоїм контролем.

Файли зберігаються як звичайні файли у звичайних теках на диску. Якщо раптом виникне потреба переїхати з DiscoDrive на інше сховище, ти просто відкриєш теку й знайдеш свої дані рівно там, де їх залишив. Жодного пропрієтарного сховища чанками, жодної бази, яку не прочитати, жодного vendor lock-in.

Кросплатформність закладена в основу: Apple, Windows, Linux та Android.

**Завантажити застосунки:** збірки для десктопа (macOS/Windows/Linux) та Android доступні на [сторінці релізів](https://github.com/kosmosoid/discodrive-apps/releases/latest); застосунок для iOS збирається з вихідного коду. Усі клієнти — у [discodrive-apps](https://github.com/kosmosoid/discodrive-apps).

## Що всередині

### 📁 Файли та синхронізація

- Швидкий сучасний **веб-файлменеджер** — завантаження, вивантаження, створення тек, перейменування, переміщення, з drag-and-drop цілих файлів *та* тек.
- **Версії** — у кожного файлу зберігається недавня історія, можна відкотити невдалу правку.
- **Кошик** — видалені файли потрапляють до кошика, звідки їх можна відновити.
- **Спільний доступ** — публічні посилання: з терміном дії, з паролем, лише на читання.
- **Підключається як диск** — відкривай сховище прямо з Finder, Провідника Windows, файлменеджера Linux чи файлового застосунку Android по WebDAV. Нічого встановлювати не потрібно.
- **Синхронізація на десктопі** — легкий застосунок тримає реальну теку на комп'ютері в синхроні із сервером, як Dropbox.
- **Дані не губляться** — якщо той самий файл змінився у двох місцях одночасно, сервер збереже обидві версії як конфліктні копії, а не мовчки перезапише одну.

### 🔒 Зашифрований сейф

- **Приватна тека з наскрізним (E2E) шифруванням** для найчутливіших файлів.
- Файли шифруються **на твоєму пристрої** ще до надсилання — сервер зберігає їх лише в зашифрованому вигляді.
- Відкривається прямо в браузері або в клієнті (розшифрування відбувається локально), а якщо забув пароль — розблоковується кодом відновлення.
- Побудований на **відкритому форматі, сумісному з [Cryptomator](https://cryptomator.org)** — якщо знадобиться відкрити сейф окремо чи перейти на інший сервіс, його можна розшифрувати в Cryptomator або іншими сторонніми утилітами.

### 📅 Календар, контакти та задачі

- Повноцінні **календарі, адресні книги та нагадування** — події, контакти й справи в одному місці.
- **Нативно на Apple** — додаєш акаунт у системних налаштуваннях iOS/macOS, і штатні Календар, Контакти та Нагадування просто працюють, без окремого застосунку.
- **На всьому іншому** — працює з DAVx⁵ на Android і з Thunderbird, Evolution чи eM Client на десктопі.
- Зручний **веб-інтерфейс** для створення та редагування подій, контактів і задач.
- **Ділися календарем або адресною книгою** з близькими.

### 🎵 Музика — твій особистий стримінг

- Перетворює твою колекцію музики на особистий стримінговий сервіс, доступний звідусіль.
- **Працює зі звичними застосунками** — будь-який OpenSubsonic-сумісний плеєр (Amperfy, Feishin та багато інших) підключається з коробки, з текстами пісень, чергою та пошуком.
- **Інтернет-радіо** — додавай улюблені станції й слухай їх через ті самі застосунки.
- **Подкасти** — підписуйся на стрічки, завантажуй і слухай епізоди, продовжуй з того місця, де зупинився, за закладками.
- **Вбудований редактор тегів** — правь назви, виконавців, альбоми, жанри й обкладинки прямо з вебу: по одному треку або цілою текою одразу.

### 📚 Книжки — твоя особиста бібліотека

- Особиста бібліотека для електронних книжок і коміксів, готова до читання на будь-якому пристрої.
- **Читається на будь-якому пристрої** — каталог за стандартом OPDS, до якого напряму підключаються KOReader, PocketBook, Marvin та інші читалки (а через браузер — узагалі будь-яка).
- **Усі потрібні формати** — EPUB, FB2, PDF, MOBI, CBZ та CBR, з мініатюрами обкладинок і пошуком по всьому каталогу.
- **Синхронізація прогресу читання** — почни книжку на одному пристрої й продовж рівно з того самого місця на іншому.
- **Редактор метаданих** — правь назви, авторів, серії, теги й описи з вебу: по одній книжці або масово по теці.

### 🛡️ Акаунти та безпека

- **Двофакторна автентифікація** TOTP через будь-який authenticator-застосунок плюс одноразові резервні коди.
- **Passkeys** — вхід за Face ID, Touch ID чи апаратним ключем, узагалі без пароля.
- **Журнал безпеки**.
- **Сповіщення на email** про важливі події безпеки й акаунта.
- **Вбудований захист від перебору паролів**.
- **Кілька користувачів** з персональними квотами.
- **Паролі для застосунків** — щоб підключати файлові, музичні та книжкові клієнти, не розкриваючи основний пароль.
- **TLS усюди** за замовчуванням.

### 🌍 Твоє, усюди, твоєю мовою

- Сумісно з операційними системами Apple, Windows, Linux та Android.
- Наразі доступні **7 мов** інтерфейсу: англійська, німецька, українська, французька, іспанська, російська та сербська.

## Встановлення

DiscoDrive — це один сервер плюс PostgreSQL. Перед ним можна поставити nginx — для TLS і швидкої роздачі файлів через X-Accel. Нижче два способи розгорнути все це. Більшості підійде **Варіант 1**.

> **Перед запуском потрібно підготувати два секретних ключі.** Сервер не запуститься з тимчасовими значеннями з `.env.example`. Згенеруй їх один раз і більше не змінюй:
>
> - `JWT_SECRET` — `openssl rand -base64 48`;
> - `SETTINGS_ENCRYPTION_KEY` — `openssl rand -hex 16`.
>
> І пам'ятай: застосунки Apple (CalDAV/CardDAV) потребують **HTTPS** — по незашифрованому HTTP вони не працюють.

### Варіант 1. Docker з клонуванням репозиторію (рекомендовано)

Найпростіший шлях: одна команда `docker compose up` піднімає сервер, базу даних і nginx з TLS.

**Потрібно:** git, Docker та Docker Compose.

```sh
git clone https://github.com/kosmosoid/discodrive.git
cd discodrive
cp .env.example .env
# відредагуй .env: задай JWT_SECRET, SETTINGS_ENCRYPTION_KEY, BASE_DOMAIN,
# POSTGRES_PASSWORD і, за бажанням, решту
docker compose up -d
```

Відкрий **https://server_address:8443** (або http://server_address:8080) — при першому запуску відкриється форма створення адміністратора. Вкажи пошту й пароль, і можна працювати. Зверни увагу: якщо плануєш налаштувати надсилання листів від сервісу, використовуй реальні поштові адреси для імен користувачів (зокрема адміністратора) — інакше листи не будуть доставлені.

Як це влаштовано:

- Файли зберігаються в `/data`.
- TLS-сертифікати — в `deploy/nginx/certs/`. Потрібно покласти туди **справжній або самопідписаний сертифікат** (як його згенерувати — описано у Варіанті 2) і налаштувати в конфігурації nginx перенаправлення з 80-го порту на HTTPS.

Щоб зупинити: `docker compose down` (дані збережуться). Щоб оновити після `git pull`: `docker compose up -d --build`.

### Варіант 2. Окремими компонентами (збірка з вихідного коду)

Якщо Docker тобі не потрібен — збери виконуваний файл самостійно й запусти його поруч зі звичайним PostgreSQL. Виконуваний файл є самодостатнім: у нього вбудовані всі необхідні компоненти.

**Для збірки потрібні:** Go 1.25+, Node.js 22+, PostgreSQL 16+.

**1. Збери веб-інтерфейс** (Nuxt → статичні файли у `web/dist`):

```sh
cd web
npm install
npm run generate
cd ..
```

**2. Збери сервер** (виконуваний файл вбере в себе `web/dist` і міграції):

```sh
CGO_ENABLED=0 go build -trimpath -o discodrive ./cmd/server
```

**3. Підніми PostgreSQL** і створи базу даних з користувачем:

```sh
createuser disco --pwprompt
createdb discodrive --owner disco
```

**4. Задай змінні оточення** (повний список — у `.env.example`):

```sh
export DATABASE_URL="postgres://disco:ПАРОЛЬ@localhost:5432/discodrive?sslmode=disable"
export JWT_SECRET="$(openssl rand -base64 48)"
export SETTINGS_ENCRYPTION_KEY="$(openssl rand -hex 16)"
export BASE_DOMAIN="example.com"
export STORAGE_ROOT="/var/lib/discodrive/data"   # тека має існувати й бути доступна для запису
export APP_PORT="8080"
export XACCEL_ENABLED="false"                     # без nginx сервер роздає файли сам
```

**5. Запусти**:

```sh
./discodrive
```

Відкрий `http://server_address:8080` і створи адміністратора.

**nginx (необов'язково).** Потрібен для HTTPS і швидкої роздачі файлів через X-Accel. Візьми за основу `deploy/nginx/default.conf`, пропиши свої TLS-сертифікати й встанови `XACCEL_ENABLED=true` — тоді сервер повертає файли заголовком `X-Accel-Redirect`, а тіло файлів роздає nginx (розташування `/data/`, значення параметра `STORAGE_ROOT`).

TLS-сертифікати візьми у Let's Encrypt (`certbot`) або згенеруй самопідписані:

```sh
openssl req -x509 -newkey rsa:2048 -nodes -days 825 \
  -keyout deploy/nginx/certs/dev-key.pem \
  -out deploy/nginx/certs/dev.pem -subj "/CN=localhost"
```

**Автозапуск через systemd (необов'язково).** Приклад модуля `/etc/systemd/system/discodrive.service`:

```ini
[Unit]
Description=DiscoDrive
After=network.target postgresql.service

[Service]
ExecStart=/usr/local/bin/discodrive
EnvironmentFile=/etc/discodrive.env
Restart=on-failure
User=discodrive

[Install]
WantedBy=multi-user.target
```

Змінні з кроку 4 поклади в `/etc/discodrive.env` (формат `KEY=value`, по одній на рядок), потім виконай:

```sh
sudo systemctl daemon-reload
sudo systemctl enable --now discodrive
```
